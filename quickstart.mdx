---
title: "Quickstart"
description: "Initialize OpenSpec in your project and run your first change in under 5 minutes."
---

<Tip>
  OpenSpec works best with high-reasoning models. Opus 4.5 and GPT 5.2 are recommended for both planning and implementation.
</Tip>

<Steps>
  <Step title="Install OpenSpec globally">
    Install the `@fission-ai/openspec` package with your preferred package manager:

    <CodeGroup>

    ```bash npm
    npm install -g @fission-ai/openspec@latest
    ```

    ```bash pnpm
    pnpm add -g @fission-ai/openspec@latest
    ```

    ```bash yarn
    yarn global add @fission-ai/openspec@latest
    ```

    ```bash bun
    bun add -g @fission-ai/openspec@latest
    ```

    </CodeGroup>
  </Step>

  <Step title="Initialize OpenSpec in your project">
    Navigate to your project directory and run `openspec init`:

    ```bash
    cd your-project
    openspec init
    ```

    This creates the following structure in your project:

    ```text
    openspec/
    ├── specs/              # Source of truth (your system's behavior)
    │   └── <domain>/
    │       └── spec.md
    ├── changes/            # Proposed updates (one folder per change)
    │   └── <change-name>/
    │       ├── proposal.md
    │       ├── design.md
    │       ├── tasks.md
    │       └── specs/      # Delta specs (what's changing)
    │           └── <domain>/
    │               └── spec.md
    └── config.yaml         # Project configuration (optional)
    ```

    - **`specs/`** — The source of truth. Describes how your system currently behaves, organized by domain (e.g., `specs/auth/`, `specs/payments/`).
    - **`changes/`** — Proposed modifications. Each change gets its own folder with all related artifacts. When complete, a change's specs merge into `specs/`.
  </Step>

  <Step title="Propose a change">
    In your AI assistant, use the `/opsx:propose` command to describe what you want to build:

    ```text
    /opsx:propose add-dark-mode
    ```

    Your AI will scaffold the change folder with all artifacts:

    ```text
    AI:  Created openspec/changes/add-dark-mode/
         ✓ proposal.md — why we're doing this, what's changing
         ✓ specs/       — requirements and scenarios
         ✓ design.md    — technical approach
         ✓ tasks.md     — implementation checklist
         Ready for implementation!
    ```

    Each change folder contains four artifacts that build on each other:

    | Artifact | Purpose |
    |---|---|
    | `proposal.md` | The "why" and "what" — captures intent, scope, and approach |
    | `specs/` | Delta specs showing ADDED/MODIFIED/REMOVED requirements |
    | `design.md` | The "how" — technical approach and architecture decisions |
    | `tasks.md` | Implementation checklist with checkboxes |
  </Step>

  <Step title="Implement with /opsx:apply">
    Once the artifacts are ready, tell your AI to implement the tasks:

    ```text
    /opsx:apply
    ```

    Your AI works through the `tasks.md` checklist and reports progress:

    ```text
    AI:  Working through tasks...
         ✓ 1.1 Created ThemeContext with light/dark state
         ✓ 1.2 Added CSS custom properties to globals.css
         ✓ 1.3 Implemented localStorage persistence
         ✓ 2.1 Created ThemeToggle component
         ...
         All tasks complete!
    ```

    If you discover the design needs adjustment during implementation, update the relevant artifact and continue.
  </Step>

  <Step title="Archive the change">
    When implementation is complete, archive the change to merge its specs into the main `specs/` directory:

    ```text
    /opsx:archive
    ```

    ```text
    AI:  Archiving add-dark-mode...
         ✓ Merged specs into openspec/specs/ui/spec.md
         ✓ Moved to openspec/changes/archive/2025-01-24-add-dark-mode/
         Done! Ready for the next feature.
    ```

    The change folder moves to `openspec/changes/archive/` for audit history, and the delta specs become part of your project's living documentation.
  </Step>
</Steps>
