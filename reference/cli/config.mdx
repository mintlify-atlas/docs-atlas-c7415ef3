title: "openspec config"

---

## Description

`openspec config` provides subcommands for reading, writing, and managing global OpenSpec settings. Configuration is stored in a YAML file whose path you can inspect with `openspec config path`.

## Syntax

```bash
openspec config <subcommand> [options]
```

## Subcommands

<AccordionGroup>
  <Accordion title="path — show config file location">
    Prints the absolute path to the global config file.

    ```bash
    openspec config path
    ```
  </Accordion>

  <Accordion title="list — show all current settings">
    Prints every key-value pair in the current global config.

    ```bash
    openspec config list
    ```
  </Accordion>

  <Accordion title="get — retrieve a specific value">
    ```bash
    openspec config get <key>
    ```

    **Example:**

    ```bash
    openspec config get telemetry.enabled
    ```
  </Accordion>

  <Accordion title="set — write a value">
    ```bash
    openspec config set <key> <value> [--string]
    ```

    Use `--string` to force the value to be stored as a string rather than being auto-typed.

    **Examples:**

    ```bash
    # Set a boolean value
    openspec config set telemetry.enabled false

    # Set a string value explicitly
    openspec config set user.name "My Name" --string
    ```
  </Accordion>

  <Accordion title="unset — remove a key">
    Removes a custom setting, reverting it to its default.

    ```bash
    openspec config unset <key>
    ```

    **Example:**

    ```bash
    openspec config unset user.name
    ```
  </Accordion>

  <Accordion title="reset — reset to defaults">
    Resets all configuration to factory defaults.

    ```bash
    openspec config reset --all --yes
    ```

    Use `--yes` to skip the confirmation prompt.
  </Accordion>

  <Accordion title="edit — open config in your editor">
    Opens the global config file in the editor defined by `$EDITOR` or `$VISUAL`.

    ```bash
    openspec config edit
    ```

    <Note>
      This is a human-only command. It requires an interactive terminal with a configured editor.
    </Note>
  </Accordion>

  <Accordion title="profile — configure workflow profile" defaultOpen={false}>
    Configure the active workflow profile interactively, or apply a preset directly.

    ```bash
    openspec config profile [preset]
    ```

    **Preset example:**

    ```bash
    # Fast preset: switch workflows to core (keeps delivery mode)
    openspec config profile core
    ```

    **Interactive wizard:**

    Running `openspec config profile` without a preset starts an action-based wizard. It first shows a summary of your current settings, then offers four choices:

    1. Change delivery + workflows
    2. Change delivery only
    3. Change workflows only
    4. Keep current settings (exit)

    <Note>
      If you choose "Keep current settings", no changes are written and no update prompt is shown.
    </Note>

    If there are no config changes but the current project files are out of sync with your global profile or delivery mode, OpenSpec shows a warning and suggests running `openspec update`.

    Pressing `Ctrl+C` cancels the flow cleanly with no stack trace and exits with code `130`.

    In the workflow checklist, `[x]` means the workflow is selected in global config. To apply those selections to project files, run `openspec update`, or choose "Apply changes to this project now?" when prompted inside a project.

    **Interactive examples:**

    ```bash
    # Delivery-only update
    openspec config profile
    # choose: Change delivery only
    # choose delivery: Skills only

    # Workflows-only update
    openspec config profile
    # choose: Change workflows only
    # toggle workflows in the checklist, then confirm
    ```
  </Accordion>
</AccordionGroup>

## All examples

```bash
# Show config file path
openspec config path

# List all settings
openspec config list

# Get a specific value
openspec config get telemetry.enabled

# Set a value
openspec config set telemetry.enabled false

# Set a string value explicitly
openspec config set user.name "My Name" --string

# Remove a custom setting
openspec config unset user.name

# Reset all configuration
openspec config reset --all --yes

# Edit config in your editor
openspec config edit

# Configure profile with action-based wizard
openspec config profile

# Fast preset: switch workflows to core (keeps delivery mode)
openspec config profile core
```
