---
title: "openspec status"
description: "Display artifact completion status for a change."
---

## Description

`openspec status` shows which artifacts in a change are complete, which are ready to work on, and which are blocked by unmet dependencies. Both humans checking progress and AI agents determining next steps use this command.

## Syntax

```bash
openspec status [options]
```

## Options

<ParamField path="--change" type="string">
  Change name to inspect. Prompts interactively if omitted.
</ParamField>

<ParamField path="--schema" type="string">
  Schema override. Auto-detected from the change's config when not specified.
</ParamField>

<ParamField path="--json" type="boolean">
  Output as JSON for agent consumption.
</ParamField>

## Examples

```bash
# Interactive status check
openspec status

# Status for a specific change
openspec status --change add-dark-mode

# JSON for agent use
openspec status --change add-dark-mode --json
```
