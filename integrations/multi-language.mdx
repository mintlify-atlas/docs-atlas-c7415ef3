---
title: "Multi-Language Support"
description: "OpenSpec works with any programming language. The spec layer is language-agnostic — examples and tooling adapt to your stack."
---

OpenSpec generates all artifacts (proposals, specs, designs, tasks) in English by default. You can switch to any language by adding a single instruction to your project config.

## Quick setup

<Steps>
  <Step title="Open your project config">
    Edit `openspec/config.yaml` in your project root.
  </Step>

  <Step title="Add a language instruction to the context field">
    Set your language in the `context` block:

    ```yaml openspec/config.yaml
    schema: spec-driven

    context: |
      Language: Portuguese (pt-BR)
      All artifacts must be written in Brazilian Portuguese.

      # Your other project context below...
      Tech stack: TypeScript, React, Node.js
    ```
  </Step>

  <Step title="Done">
    All generated artifacts will now be written in your specified language.
  </Step>
</Steps>

## Tips

### Handle technical terms

Decide how to handle technical terminology. You can instruct the AI to keep specific terms in English:

```yaml
context: |
  Language: Japanese
  Write in Japanese, but:
  - Keep technical terms like "API", "REST", "GraphQL" in English
  - Code examples and file paths remain in English
```

### Combine with other context

Language settings work alongside the rest of your project context — just include both in the same `context` block:

```yaml openspec/config.yaml
schema: spec-driven

context: |
  Language: Portuguese (pt-BR)
  All artifacts must be written in Brazilian Portuguese.

  Tech stack: TypeScript, React 18, Node.js 20
  Database: PostgreSQL with Prisma ORM
```

## Verification

To confirm your language configuration is active, inspect the generated instructions for a change:

```bash
# Check the instructions — should show your language context
openspec instructions proposal --change my-change

# Output will include your language context
```

If the output includes your language instruction, all artifacts for that change will be generated in the specified language.
